Help & Information

Accuracy Disclaimer

This Mapping application shows a summary of information gleaned from public squid proxies.
The information collected reflects actual requests by end-users utilizing these open-proxies to browse to well known websites.
However, this information is not complete. For example, if a deployment location or customer domain does not show up on this list; it does not indicate there is no such occurance of either item.

More information about the source data can be found on Sean Butler's page.

Main View: Competition Stats

The main view shows a summary of all available companies with the number visible regions as well as seen customer domains.
Regions are identified as a unique geographical locality and may contain one or more IP address. (eg: San Francisco, CA, USA)
Domains are identified by the second level domain name and may contain one or more sub-domains. (eg: akamai.com)
Clicking on the value-link will display a corresponding map or list.

Deployment Views

Deployment Map

The deployment map view will display a Google Map with colored markers identifying the geographical location of each known deployment. A number will be shown in each marker indicating the number of IP address' seen.
A table below the map will summarize the information:
  • # of IP address' seen at the location
  • Sample IP address
  • AS Network name
  • Geographical location
  • Date the loctaion was last seen

Deployment List

Viewing the deployment IP list will display all known IP address for a given selection.
Entries which are no longer visible or which have changed locations will be struck out.
The following information is displayed in the table.
  • IP address
  • AS Network name
  • Geographical location
  • Date the loctaion was last seen

Customer Views

Customer Domains

This view will show all domains for a given selection grouped by the 1st two components of a domain. (eg: akamai.com)
Entries which no longer appear for a selected company will be shown with a strike-out line. The following information is displayed
  • # of sub-domains
  • Customer domain name
  • Date this record was last seen

Customer Hosts

The Customer Hosts view is similar to the domain view with the exception that ALL hosts are displayed.
Clicking on one of the host-name entries will open that URL in a new browser window.
  • Customer domain name
  • Date this record was last seen

Searching

Any type of search may be done using standard or MYSQL wildcards:
  • '*' & '?' will match 'anything' and 'a character' respectively
  • '%' & '_' are MYSQL equivalents

Deployments by geography

Generally a search by geography may be done by entering localities in the form of:
  [City] , [State] , [Country]
Where items are separated by a comma (,)
Combinations of two items (eg: A,B) will match either City,State or State,Country.

Customers by domain name

Searching by domain name follows a similar pattern to that of geography. Use the form of:
  [4th level] . [3rd level] . [2nd level] . [1st level]
Where items are separated by a period (.)
Combinations of multiple items will be matched against all posibilities.

Examples

  • * = match anything
  • amazon.com = match exactly 'amazon.com'
  • *amazon = match anything with 'amazon' in it. (eg: one.amazon.com, wildamazon, etc.)
  • ca = match CA(lifornia) or (CA)nada
  • ca,us = match only CA(lifornia), (US)A
  • us = match only (US)A
